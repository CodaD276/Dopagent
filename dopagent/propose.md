# Propose Template
# 把任务包装成有吸引力的提议

## 核心原则

不是 "你应该做 X"。
是 "X 这件事，以你现在的状态，大概率会爽。"

## 模板

> "[任务描述]，而且 [为什么现在是好时机]。
>  我翻了一下 [经验库/热存储]，[相关历史证据]。
>  [预估耗时]，[预计爽感来源]。"

## 示例

坏："你应该把草莓 EC 值的记录写了。"

好："你上次说草莓 EC 值达标了，记录还挂着。而且今晚刚跑通 Alaya，
     状态正好。大概五分钟的事，写完这条你这周的农业项目就闭环了。"

## 模式要素

| 要素 | 来源 | 示例 |
|---|---|---|
| 任务描述 | 热存储/待办 | "EC值记录还没写" |
| 时机判断 | Dopagent profile | "今晚状态正好" |
| 历史证据 | 经验库 | "上次你完成类似 micro-task 后情绪回升" |
| 预估耗时 | 微步骤分解 | "五分钟" |
| 爽感锚点 | 经验库 | "这条写完农业项目闭环" |

## 反模式

- "你需要..." ← 命令式
- "建议你..." ← 疏离
- "要不要..." ← 太弱，没有推动力
- 在 recovery 模式下不提任何任务 ← 只提供选项，不推动
